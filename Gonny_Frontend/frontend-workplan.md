# Gonny 프론트엔드 작업 정리

## 1. 프로젝트 한 줄 요약

`Gonny`는 카테고리 선택 + 자연어 입력으로 AI 여행 일정을 생성하고, 여행 중 실시간 수정/체크인/예산 관리/공유/회고까지 지원하는 모바일 우선 반응형 웹 서비스다.

## 2. 프론트엔드 핵심 사용자 흐름

1. 로그인한다.
2. 여행 조건을 입력해 AI 일정을 생성한다.
3. 생성된 일정을 일자별 카드/타임라인/지도 기준으로 확인한다.
4. 날씨, 동행자, 숙소, 예산, 장소 대체 추천 등으로 일정을 조정한다.
5. 여행 중 체크인과 지출 입력을 한다.
6. 일정 PDF를 받거나 공유 링크를 만든다.
7. 여행 종료 후 회고 리포트를 확인한다.

#### 1) 랜딩 / 온보딩
- 서비스 소개
- "여행 일정 만들기" CTA
- 계절 추천 여행지 / 축제 / 환율 / 인기 여행지 진입점
- 비로그인 상태에서도 일부 탐색 가능

#### 2) 로그인
- 카카오 로그인 버튼
- 구글 로그인 버튼
- 로그인 성공 후 토큰 저장
- 신규 사용자면 프로필 설정 화면으로 이동

#### 3) 프로필 / 계정 설정
- 닉네임
- 기본 여행 스타일 태그
- 기본 동행자 유형
- 내 여행 히스토리 목록
- 로그아웃
- 회원 탈퇴

### B. 여행 생성

#### 4) 여행 생성 폼
- 여행지
- 여행 기간
- 여행 스타일 복수 선택
- 예산 / 통화
- 이동 수단
- 숙박 유형
- 식사 스타일
- 동행자 입력
- 기타 요청 자연어 입력

필수 UX:
- 카테고리 먼저 선택하고 자연어는 보조 입력으로 항상 열려 있어야 함
- 모바일에서 한 화면에 다 때려 넣기보다 섹션 스텝형 UI가 적합
- 생성 중 스트리밍 상태 표시 필요
- 무료 플랜 월 3회 제한 에러 노출 필요

#### 5) 생성 중 / 결과 준비 화면
- 스트리밍 진행 상태
- 생성 중 문구
- 실패 시 재시도
- 부분 결과가 있으면 일자별 skeleton 또는 placeholder로 먼저 보여주기

### C. 여행 상세

#### 6) 내 여행 목록
- 최신순 카드 리스트
- 여행지 / 기간 / 상태 / 썸네일
- 상태값 예시: `upcoming`, `ongoing`, `completed`
- 페이지네이션

#### 7) 여행 상세 메인
- 여행 제목
- 여행 기간 / 여행지 / 예산 요약
- 날씨 요약
- 공유 / PDF / 삭제 / 기본 정보 수정 액션
- 탭 권장:
  - 일정
  - 지도
  - 숙소
  - 동행자
  - 예산
  - 회고

#### 8) 일자별 일정 탭
- Day 단위 아코디언 또는 탭
- 각 일정 아이템 카드:
  - 장소명
  - 장소 유형
  - 시작 시간
  - 체류 시간
  - 이동 시간
  - 이동 수단
  - 주소
  - 예상 비용
  - AI 팁
  - 체크인 상태
- 하루 총 예상 시간 표시
- 10시간 초과 시 과부하 경고 배너

#### 9) 장소 수정 / 대안 추천 모달
- "이 장소 바꾸기"
- 카테고리 선택
- 추가 요청 입력
- 기존 시간 유지 여부
- 대안 3개 비교 카드
- 수정 전/후 비교 UI
- 되돌리기 히스토리 표시

#### 10) 날씨 경고 / 대안 일정 추천
- 특정 날짜 상단에 날씨 아이콘과 기온 노출
- 비/눈 예보 시 경고 카드
- "원래 일정 유지"
- "AI 대안 보기"
- 대안 코스 3개 카드
- 선택 적용 전 미리보기

#### 11) 체크인 흐름
- 각 일정 카드에 "여기 도착했어요" 버튼
- 위치 권한 요청
- 성공 시 완료 처리
- 다음 일정 하이라이트
- 장소 팁 토스트 또는 바텀시트 노출

### D. 부가 기능

#### 12) 숙소 추천 탭
- 숙소 추천 목록 3~5개
- 필터:
  - 숙소 타입
  - 최대 가격
- 숙소 카드:
  - 이름
  - 유형
  - 가격대
  - 특징 태그
  - 체크인/체크아웃
  - 예약 링크
- 지도 마커 연동

#### 13) 동행자 관리 탭
- 동행자 최대 4명 입력
- 이름
- 취향 태그 복수 선택
- 아이 동반 여부
- 저장 후 반영 요약 노출

#### 14) 예산 관리 탭
- 총 예산 / 총 지출 / 잔여 예산
- 사용률 게이지
- 카테고리별 지출 차트
- 지출 입력 폼:
  - 카테고리
  - 금액
  - 통화
  - 메모
  - 지출 시각
- 초과 예산 경고

#### 15) 공유 / PDF
- PDF 다운로드 버튼
- 공유 링크 생성 모달
- 읽기 전용 / 편집 가능 선택
- 만료 기간 선택
- 생성된 URL 복사

#### 16) 공유 일정 공개 화면
- 비로그인 접근 가능
- 일정 요약 및 일자별 보기
- 권한에 따라 읽기 전용 또는 편집 UI 분기 필요

#### 17) 회고 리포트
- 총 지출
- 예산 대비 차이
- 방문 완료 장소 수
- 총 이동 거리
- 카테고리별 지출 차트
- AI 인사이트 3개 이상
- 리포트 PDF 열기

### E. 탐색형 화면

#### 18) 시즌 추천 / 축제 / 영감 피드
- 계절별 추천 여행지

## 4. API 기준 화면 매핑

| 화면/기능 | API |
| --- | --- |
| 카카오 로그인 | `POST /auth/kakao` |
| 구글 로그인 | `POST /auth/google` |
| 토큰 재발급 | `POST /auth/refresh` |
| 프로필 조회/수정/탈퇴 | `GET/PATCH/DELETE /users/me` |
| 여행 생성 | `POST /trips` |
| 내 여행 목록 | `GET /trips` |
| 여행 상세 | `GET /trips/{trip_id}` |
| 여행 기본 정보 수정 | `PATCH /trips/{trip_id}` |
| 여행 삭제 | `DELETE /trips/{trip_id}` |
| 장소 수정 | `PATCH /trips/{trip_id}/items/{item_id}` |
| 대안 장소 추천 | `POST /trips/{trip_id}/items/{item_id}/alternatives` |
| 체크인 | `POST /trips/{trip_id}/items/{item_id}/checkin` |
| 날씨 조회 | `GET /weather/{destination}` |
| 날씨 기반 대안 일정 | `POST /trips/{trip_id}/weather-recommend` |
| 숙소 추천 조회/선택 저장 | `GET/POST /trips/{trip_id}/accommodations` |
| 동행자 등록/조회/삭제 | `POST/GET/DELETE /trips/{trip_id}/companions` |
| 지출 입력/목록/삭제 | `POST/GET/DELETE /trips/{trip_id}/expenses` |
| 예산 요약 | `GET /trips/{trip_id}/expenses/summary` |
| 일정 PDF | `GET /trips/{trip_id}/pdf` |
| 공유 링크 생성 | `POST /trips/{trip_id}/share` |
| 비로그인 공유 화면 | `GET /share/{token}` |
| 회고 리포트 | `GET /trips/{trip_id}/report` |
| 회고 PDF | `GET /trips/{trip_id}/report/pdf` |
| 시즌 추천 | `GET /seasons/recommend` |
| 축제 캘린더 | `GET /seasons/festivals` |
| 환율 | `GET /exchange-rate` |

## 10. 화면 설계 시 주의할 점

- 모바일 우선이어야 한다.
- AI 일정 생성은 최대 30초까지 걸릴 수 있으므로 대기 경험 설계가 중요하다.
- 장소 수정은 날짜 전체 재생성이 아니라 "아이템 단위 수정"이라는 차별점을 UI에서 강하게 보여줘야 한다.
- 날씨는 자동 반영이 아니라 추천 후 사용자가 선택하는 흐름이어야 한다.
- 체크인은 위치 권한 실패, GPS 오차 500m 초과 같은 실패 케이스 UI가 반드시 필요하다.
- 예산은 숫자만 보여주지 말고 게이지와 카테고리별 시각화가 필요하다.
- 공유 화면은 비로그인 접근 가능하므로 인증 라우트와 분리해야 한다.

## 11. 프론트에서 먼저 확인할 백엔드 미정 사항

- `GET /trips/{trip_id}` 응답 스키마 상세
- `PATCH /trips/{trip_id}` 요청/응답 스키마
- `GET /trips/{trip_id}/expenses` 응답 형식
- `POST /trips/{trip_id}/accommodations` 저장 형식
- `GET /trips/{trip_id}/companions` 응답 형식
- `GET /trips/{trip_id}/report/pdf` 응답 형식
- 여행 생성 스트리밍 응답 포맷이 `SSE`인지 `chunked json`인지 여부
- 공유 링크의 `edit` 권한일 때 실제 허용 범위
